Módulo 1: Git & GitHub

El estándar en control de versiones

¿Qué es Git?

Es un sistema de **control de versiones** de código. Permite determinar puntos de guardado distribuidos a lo largo del desarrollo.

¿Por qué usar Git?

  • Permite guardar y retroceder el estado del código a un punto anterior (**Commit**).
  • Facilita el trabajo colaborativo en paralelo mediante ramas (**Branches**).
  • Gracias a GitHub, puedes respaldar tus repositorios de forma remota y segura.

Preparación

Asegúrate de tener instalados tus entornos de trabajo:

Link tutorial de Descarga para VSCode y Git

Además, es un requisito obligatorio crearse una cuenta en **GitHub** para las prácticas.

Comandos Principales

Trabajaremos directamente desde la terminal de Git Bash

Configuración Básica

Establece tu identidad global en la máquina:


                    $ git config --global user.name "Tu_Usuario"
                    $ git config --global user.email correo@ejemplo.com

Inicio y Clonación

Inicializar un repositorio local en la carpeta actual:


                    $ git init

Clonar un repositorio remoto existente:


                    $ git clone <url>

Flujo de Trabajo diario

Gestión y confirmación de cambios locales

Ciclo Básico de un Cambio

Ver el estado actual de los archivos (modificados o nuevos):


                    $ git status

Añadir los archivos modificados al área de preparación (Staging Area):


                    $ git add .

Confirmación y Envío

Confirmar los cambios en el historial local con un mensaje descriptivo:


                    $ git commit -m "Mensaje descriptivo del commit"

Subir tus commits locales a la rama remota correspondiente:


                    $ git push origin main

Actualización remota

Traer y fusionar los cambios más recientes del servidor a tu copia local:


                    $ git pull origin <nombre_rama>

Historial e Inspección

Ver cómo ha evolucionado el repositorio

Inspeccionar Commits

Mostrar el historial completo de commits:


                    $ git log

Historial simplificado en una línea y con gráfico de ramas:


                    $ git log --oneline --graph

Ver diferencias

Ver las modificaciones específicas realizadas en los archivos antes de confirmar:


                    $ git diff

El código ya no funciona :( ¿Que hago?

Opción A: Volver atrás sin borrar el historial (Crea un nuevo commit que deshace los cambios del commit indicado):


                    $ git revert <id_commit>

Opción B: Volver atrás borrando de raíz (Mueve el estado actual al commit indicado, eliminando los commits posteriores):


                    $ git reset --hard <id_commit>

El reset '--hard' destruye los cambios no guardados de forma permanente.

Gestión de Ramas (Branches)

Trabajando en múltiples características en paralelo

Comandos de Ramas

Listar todas las ramas locales existentes:


                    $ git branch

Crear una nueva rama independiente:


                    $ git branch <nombre_rama>

Navegación y Fusión

Cambiar tu espacio de trabajo a la rama especificada:


                    $ git checkout <nombre_rama>

Fusionar una rama específica con tu rama activa actual:


                    $ git merge <nombre_rama>

Conexión de Repositorios Remotos

Vincular un repositorio local a un servidor remoto por primera vez:


                    $ git remote add origin <url>

Listar los alias remotos configurados junto con sus URLs de conexión:


                    $ git remote -v

Algunas buenas practicas de los commits

Para poder organizarce bien, es recomendable seguir unas reglas con los commits

Iniciarlos con un verbo:

  • feat: Nueva caracteristica
  • fix: bug areglado
  • perf: mejora de rendimiendo
  • docs: Cambio en la documentación
  • refactor: Refactorización del código

Otras cositas más:

  • No usar puntos finales o suspensivos
  • Crear commits cortos y consisos
  • Referenciar issues y pull requests
